Introducción

Introduccion a los terminos programacion orientada a objetos (POO) asi como la programacion orientada a eventos (POE) tambien se explicara que es la programacion procedimental.

Programación orientada a eventos.

La programación orientada a eventos (POE) es un paradigma de programación en el cual el flujo de ejecución del programa se basa en la ocurrencia de eventos. Un evento es cualquier suceso que ocurra en el sistema, como un clic del ratón, una pulsación de tecla, la recepción de un mensaje, una señal de temporizador, etc.

En la POE, se definen los eventos que pueden ocurrir en el sistema y se especifica cómo el programa debe responder a cada uno de ellos. En lugar de tener un flujo de ejecución lineal y predecible, como en la programación estructurada, la POE permite que el programa responda de forma dinámica a los eventos que ocurran.

Los lenguajes de programación que admiten la POE suelen tener un modelo de ejecución basado en bucles de eventos, donde el programa espera continuamente la ocurrencia de un evento y, cuando ocurre, ejecuta una función de respuesta específica para ese evento.

La POE es ampliamente utilizada en el desarrollo de aplicaciones de usuario, interfaces gráficas de usuario (GUI), juegos, sistemas de red y dispositivos embebidos, entre otros. Ejemplos de lenguajes de programación que admiten la POE son JavaScript, Python, C#, Java, entre otros.

Lenguajes procedimentales.

La programación procedimental es un paradigma de programación que se centra en la creación de procedimientos o funciones que realizan tareas específicas y que se llaman desde el programa principal para realizar una tarea determinada.

En la programación procedimental, el programa se divide en una serie de procedimientos o funciones que se ejecutan secuencialmente, con la posibilidad de pasar datos entre ellas. Cada procedimiento realiza una tarea específica, y los datos de entrada y salida se definen claramente.

La programación procedimental se basa en la idea de que el programa es una serie de instrucciones que se ejecutan secuencialmente, donde cada instrucción se lleva a cabo de manera lineal. El programa principal llama a los procedimientos o funciones según sea necesario para realizar una tarea específica.

Este enfoque se utiliza comúnmente en lenguajes de programación como C, Pascal y Fortran. La programación procedimental es útil para aplicaciones en las que se requiere un alto rendimiento y control cercano del hardware, como en sistemas operativos, controladores de dispositivos, aplicaciones científicas y de ingeniería, entre otros.

A diferencia de la programación orientada a objetos, que se centra en objetos y clases, la programación procedimental se centra en procedimientos o funciones que se utilizan para realizar tareas específicas. Aunque la programación orientada a objetos se ha convertido en un enfoque más popular en la programación moderna, la programación procedimental todavía se utiliza ampliamente y es una base importante para muchos sistemas y aplicaciones.
